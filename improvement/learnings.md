# learnings.md — 指摘ログ（承認待ちキュー＋確定ルール）

> SPEC.md §4 の中核ファイル。「使うほど会社専用に賢くなる」蓄積先。
> セッション開始時に読み、終了時に更新する（learnings loop パターン）。
> 運用手順は `improvement/SKILL.md`（translation-improve スキル）を参照。

---

## このファイルの使い方（要点）

- **auto-memory** = セッション内の揮発的な気づき（下書き）。正本ではない。
- **本ファイル** = 承認を待つ正式キュー＋確定ルールの蓄積先（git 管理の正本）。
- フロー: 気づき → §A 承認待ちキューへ → 承認 → §B 確定ルールへ昇格 → 該当 `profile-*/SKILL.md` または `common/house-style.md` へ反映。
- **記録時は必ず固有名詞をマスキング**（`common/house-style.md` §1 / SPEC §3.5）。

### 昇格の二経路（SPEC §4.1）
- **主経路**: 人が即承認すれば 1 件でも昇格（運用初期はこれが基本）。
- **安全弁**: 人の承認がないまま同種が N 件溜まったら自動で「ルール化候補」として提案（過剰一般化防止）。

---
