# 多ジャンル翻訳支援システム

翻訳・英訳の修正・ナレーション原稿の推敲を、**ジャンルに合わせて**手伝う Claude Code 用のツールです。
使うほど、あなたの会社のやり方（よく直す点・定訳）を覚えていきます。

---

## できること

原稿を渡すと、まず種類（音声台本／広報・読み物／学術・申請／契約・公文書）を見分け、
それぞれに合った基準で翻訳・推敲します。あなたの直しを覚えて、次回から活かします。

- **音声台本**: 声に出して自然か（尺・息継ぎ・読み）を重視
- **広報・読み物**: 自然な日本語/英語に。AI っぽい文を取り除く
- **学術・申請**: 主張や定訳を勝手に変えない。忠実さ最優先
- **契約・公文書**: 逐語。改変しない

## 導入（ダウンロードして開くだけ）

1. このフォルダを入手する（どれか1つ）

   ```bash
   # git がある場合（あとで更新を取り込める）
   git clone https://github.com/rozwer/translation-skills.git

   # git がない場合：zip を落として解凍
   curl -L -o translation-skills.zip \
     https://github.com/rozwer/translation-skills/archive/refs/heads/main.zip
   unzip translation-skills.zip
   ```

   ブラウザで https://github.com/rozwer/translation-skills を開き、緑の「Code」→「Download ZIP」でも入手できます。

2. 解凍した**フォルダを Claude Code（Claude Desktop アプリの「Code」タブ）で開く**。
3. これで準備完了です。特別な設定はいりません。

---

## 使い方（毎回の流れ）

1. 入力欄に **`/translate`** と打って送る（このあと原稿を貼り付けてもよいし、`/translate` のあとに原稿を続けて書いてもよい）。
2. いくつか質問されます（ジャンル／どこまで忠実に／字数・締切／機密かどうか）。**選ぶだけ**でかまいません。
3. 結果を確認し、直したいところを伝えます。
4. 作業の終わりに **「改善ループを起動して」** と言うと、あなたの直しを“会社のルール”として覚えるか、**確認画面（差分）で聞いてきます**。OK したものだけ覚えます。

> `/translate` が入口です。入力欄に `/` を打つと候補に出てきます。
> （打ち忘れても「翻訳して」と頼めば動きますが、毎回 `/translate` から始めるのが確実です。）
> ジャンルの判断は、あなたの選択がそのまま使われます。Claude が勝手に決めることはありません。

---

## 守ってほしいこと（機密）

- クライアントの**実名・社名は、記録するとき Claude が自動で伏せ字**（`<CLIENT_A>` など）にします。確認画面で伏せ忘れがないか見てください。
- **預かった原稿そのものは、共有フォルダや公開の場所に置かない**でください。
- 機密の案件は「**これは機密です**」と一言伝えてください。扱いが慎重になります。

---

## うまくいかないときは

- ジャンルを選び直したいとき → 「ジャンルを選び直したい」と伝える。
- 直してほしい点が反映されないとき → 具体的に「ここをこう」と伝える。覚えさせたいなら「これはルールにして」と言う。
- 動きがおかしいとき → このフォルダを最新版に入れ替える（上の「導入」をやり直す）。
