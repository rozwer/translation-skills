# 音声台本プロファイル

ナレーション原稿・アナウンス・CMコピーなど、声に出して使われる成果物のためのプロファイル。
横断ルールと機密マスキング（§3.5）は `common/house-style.md` に従う。ここではこのジャンル固有の規則だけを持つ。

## 忠実さ基準

効果・意図への忠実を最優先する。逐語性より「聞き手にどう届くか」を優先し、transcreation（意訳・再創作）を許容する。

- 直訳より、ターゲット言語で自然に聞こえる言い回しを採る。
- 元原稿の情報・トーン・コールトゥアクションは保持する。意図を変える改変はしない。
- 固有名詞・数値・日時・金額など事実情報は動かさない。
- 尺（秒数・文字数）の制約があれば、それを満たす範囲で言い換える。

## 使用する書き換えツール

- `blader/humanizer`: 声・personality・soul を足す。台本に温度を与える主力ツール。
- 採否と配線は `tools/ADOPTION.md`（§8.1 判定テーブル）に従う。未検証ツールは本流に入れない。
- 検出系（§6）はこのプロファイルでは基本不要。回避目的の書き換えは行わない。

## チェック手順

自動化を厚めに、人間レビューは軽めにかける。最後に必ず「声に出して読めるか」を確認する。

1. ルーターから受けた尺・トーン・媒体（ラジオ／映像／館内放送等）の制約を確認する。
2. transcreation で訳出・推敲し、`blader/humanizer` で声を足す。
3. **音読チェック（このプロファイルの要）**
   - 尺: 想定秒数・話速で収まるか（早口にならないか）。
   - 息継ぎ: 一息で読めない長文がないか。読点・文の切れ目が自然か。
   - 難読語: 読み間違えやすい・噛みやすい語がないか。同音異義で誤解されないか。
   - 数字の読み: 「2024」「3,500円」等が意図どおりに読まれるか。必要なら読み仮名・分割を付す。
4. 事実情報（固有名詞・数値・日時）が原稿と一致するか照合する。
5. 機密度タグが「機密」の案件は、learnings/glossary へ上げる前に `common/house-style.md` §3.5 のマスキングを通す。

## NG集（seed）

明らかな一般則のみ。クライアント固有・媒体固有の細目は TODO へ。

| NG | 直し |
|---|---|
| 一息で読めない長文（読点なしで続く） | 文を分割し、息継ぎ位置を作る |
| 黙読前提の書き言葉（硬い漢語の連続） | 耳で分かる平易な語に置換 |
| 読みが一意に定まらない数字・略語 | 読み方を確定（読み仮名・分割・スペルアウト） |
| 同音異義で誤解を招く語 | 別語に言い換える |
| 直訳由来の不自然な語順・冗長表現 | ターゲット言語で自然な言い回しへ |

## 境界ケースの扱い

- 台本と読み物の中間（Web記事を音声化する等）: 無理に片方へ寄せず、不確実性を明示フラグとして残し、`common/house-style.md` の境界ケース規則で処理する。
- 尺と忠実性が両立しないとき: どちらを優先するかをルーターの制約（設問3）で確認し、判断をノートに残す。
- 字幕・テロップ併用など読み上げない要素が混在する場合は、音読チェックの対象外として切り分ける。
