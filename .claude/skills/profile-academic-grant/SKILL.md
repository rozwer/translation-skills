# 学術・申請プロファイル

論文・科研申請書・要旨・研究計画書など、命題の正確さと定訳が命の成果物のためのプロファイル。
横断ルールと機密マスキング（§3.5）は `common/house-style.md` に従う。ここではこのジャンル固有の規則だけを持つ。

## 忠実さ基準

命題への高忠実。自然さより忠実性を優先する。次の3点を絶対に守る。

- **claim の強弱を動かさない**: 「示唆する／示す／証明する」等の確信度・限定（may, suggest, demonstrate 等）を勝手に上げ下げしない。
- **定訳から外れない**: 分野の確立した訳語・専門用語・固有表現に従う。気の利いた言い換えをしない。
- **存在しない参照を入れない**: 引用・文献・数値・図表番号を捏造・補完しない。原文にあるものだけを訳す。

その他、固有名詞・数値・単位・統計量は逐一照合する。

## 使用する書き換えツール

- `matsuikentaro1/humanizer_academic`: formal な格を壊さずに学術 AI 臭のみを除く。formality を下げる方向には使わない。
- 採否と配線は `tools/ADOPTION.md`（§8.1 判定テーブル）に従う。
- 検出系（§6）は**本流に組み込まない**（§6.3）。検出を通すための書き換えは忠実性を削り、申請書では致命的。

## チェック手順

人間レビューは**必須**。自動化は浅く（下訳＋整合まで）にとどめ、最終判断は人が行う。

1. ルーターから受けた分野・定訳の有無・フォーマット要件（文字数・様式）を確認する。
2. 下訳を作る。定訳・専門用語は用語集（glossary）と照合し、確定訳に揃える。
3. **忠実性チェック（このプロファイルの要）**
   - claim の確信度・限定が原文と一致するか。
   - 用語が定訳から外れていないか。
   - 引用・文献・図表番号・数値・統計量が原文と一致し、捏造がないか。
4. `humanizer_academic` で AI 臭のみを除く（格・formality は保持）。
5. **人間レビュー必須**: 訳者が命題の正確さを確認して承認する。自動確定しない。
6. 機密度タグが「機密」の案件は、learnings/glossary へ上げる前に `common/house-style.md` §3.5 のマスキングを通す。

## NG集（seed）

明らかな一般則のみ。分野固有の定訳・細目は TODO へ。

| NG | 直し |
|---|---|
| claim の確信度を勝手に変更（suggest→show 等） | 原文の確信度に厳密に合わせる |
| 定訳を無視した独自訳・言い換え | 確定訳に戻す |
| 原文にない引用・文献・数値の補完 | 削除。原文にあるものだけ訳す |
| 限定詞・条件節（may, under X 条件）の脱落 | 原文どおり訳出する |
| 自然さのための意訳で命題がずれる | 逐語性を優先して直す |

## 境界ケースの扱い

- 学術と広報の中間（プレスリリース化された研究紹介）: 忠実性側を既定とし、自然さを足すなら命題を動かさない範囲に限る。不確実性は明示フラグとして残す。
- 定訳が複数あり確定できない場合: 自動で選ばず、候補をノートに挙げて人の判断を仰ぐ。
- 検出スコアが高くても、忠実性を削る書き換えはしない（§6.3）。判断は `common/house-style.md` の境界ケース規則による。
