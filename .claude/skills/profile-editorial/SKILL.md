# 広報・読み物プロファイル

記事・ブログ・社内報・PR文など、読んで伝わることが目的の成果物のためのプロファイル。
横断ルールと機密マスキング（§3.5）は `common/house-style.md` に従う。ここではこのジャンル固有の規則だけを持つ。

## 忠実さ基準

自然さ寄り。読みやすく自然な文章を優先するが、事実は固定する。

- 言い回し・構成は自然さのために調整してよい。
- 固有名詞・数値・日時・引用・出典など事実情報は改変しない。
- 主張の強弱・ニュアンスは原文の範囲に収める。誇張・脚色を足さない。
- AI 臭・機械翻訳調を徹底的に除く（このプロファイルの主目的）。

## 使用する書き換えツール

- `conorbronsdon/avoid-ai-writing`: AI 文体の徹底クリーニング。英語成果物の主力。
- 日本語成果物には `avoid-ai-writing-ja`（採用判定中・§H1）。英語特化ツールを日本語にそのまま当てない。
- 採否と配線は `tools/ADOPTION.md`（§8.1 判定テーブル）に従う。

## チェック手順

人間レビューは中程度。検出器（§6）は必要時のみ手動で、合否判定ではなくリスク確認に使う。

1. ルーターから受けた媒体・読者層・トーンの制約を確認する。
2. 訳出・推敲し、書き換えツールで AI 臭をクリーニングする。
3. 事実情報（固有名詞・数値・引用・出典）が原文と一致するか照合する。
4. **（任意）検出器によるリスク確認**（§6.2）
   - 自動実行はしない。納品前に担当者が手動でコピペ実行する。
   - スコア・使用ツール・日時を訳者ノートに記録する。これが人間関与の記録（§6 防御線）になる。
   - スコアは合否判定ではない。低スコアでも「検出を通すための書き換え」はしない。
5. 機密度タグが「機密」の案件は、learnings/glossary へ上げる前に `common/house-style.md` §3.5 のマスキングを通す。

## NG集（seed）

明らかな一般則のみ。クライアント固有・媒体固有の細目は TODO へ。

| NG | 直し |
|---|---|
| AI 定型表現（「いかがでしたか」「〜と言えるでしょう」「ポイントは3つ」） | 内容に即した具体的な記述へ |
| 中身のない前置き・要約の重複 | 削るか、情報を足す |
| 体言止めの連続・同一構文の反復 | 文末・構文を変化させる |
| 直訳由来の不自然な語順・冗長な修飾 | 自然な言い回しへ整える |
| 原文にない誇張・断定の追加 | 原文の強弱の範囲に戻す |

## 境界ケースの扱い

- 広報と台本の中間（読み上げ前提のPR文等）: 不確実性を明示フラグとして残し、音読チェックが要るなら audio-script の基準を併用する。判断は `common/house-style.md` の境界ケース規則による。
- 広報と学術の中間（技術記事・ホワイトペーパー）: 事実・定訳の比重が高ければ忠実性側に寄せ、判断をノートに残す。
- 検出スコアと自然さが衝突する場合は、自然さ（品質）を優先し、検出は記録にとどめる。
