# 契約・公文書プロファイル

契約書・約款・規程・公的文書など、逐語性と改変禁止が命の成果物のためのプロファイル。
横断ルールと機密マスキング（§3.5）は `common/house-style.md` に従う。ここではこのジャンル固有の規則だけを持つ。

## 忠実さ基準

逐語・改変厳禁。このプロファイルでは自然さ・読みやすさのための裁量を認めない。

- 原文の構造・条項・順序・番号付けを保持する。文の統合・分割・並べ替えをしない。
- 法律用語・定義語は確定訳で固定する。同一語は同一訳で一貫させる（訳ゆれ禁止）。
- 数値・期間・金額・当事者名・条項参照を逐一保持する。丸めや言い換えをしない。
- 義務・権利・条件（shall / may / must / 〜しなければならない等）の強弱を変えない。
- 曖昧・誤りに見える箇所も勝手に直さない。疑義はノートに記し人に上げる。

## 使用する書き換えツール

- **書き換え系は当てない（禁忌）**。AI 臭除去・humanize も逐語性を壊すため使わない。
- 使うのは detect / 監査のみ（差分確認・訳ゆれ検出・整合チェック）。
- 採否と配線は `tools/ADOPTION.md`（§8.1 判定テーブル）に従う。
- 検出系（§6）は**本流に組み込まない**。逐語性が最優先で、検出を通すための書き換えはしない。

## チェック手順

人間レビューは**必須**。自動化は最小（用語照合・整合チェックの補助まで）。

1. ルーターから受けた文書種別・定義語・参照規約・様式を確認する。
2. 逐語訳を作る。定義語・法律用語は用語集（glossary）の確定訳で固定する。
3. **逐語性・整合チェック（このプロファイルの要）**
   - 条項・番号・順序が原文と一致するか。
   - 同一語が同一訳になっているか（訳ゆれ検出）。
   - 数値・期間・金額・当事者名・条項参照が原文と一致するか。
   - 義務・権利・条件の強弱が原文どおりか。
4. 監査（detect / 差分）で改変・脱落がないかを機械的に確認する。書き換えはしない。
5. **人間レビュー必須**: 訳者が逐語性を確認して承認する。疑義は直さずノートに記す。
6. 機密度タグが「機密」の案件は、learnings/glossary へ上げる前に `common/house-style.md` §3.5 のマスキングを通す。

## NG集（seed）

明らかな一般則のみ。文書種別・法域固有の細目は TODO へ。

| NG | 直し |
|---|---|
| 自然さのための文の統合・分割・並べ替え | 原文の構造・順序を保持 |
| 同一語の訳ゆれ | 確定訳で統一 |
| 義務・条件の強弱変更（shall→may 等） | 原文どおりの強さに戻す |
| 数値・期間・金額・固有名詞の丸め・言い換え | 原文どおり逐語保持 |
| 曖昧箇所の独断修正・補完 | 直さずノートに疑義として記す |

## 境界ケースの扱い

- 契約と公的広報の中間（規約の説明文等）: 逐語が要る本文と説明文を切り分け、本文は本プロファイル、説明文は editorial へ振る。不確実性は明示フラグとして残す。
- 原文に誤記・矛盾がある場合: 勝手に直さず、訳注・ノートで人に上げる。
- 判断は `common/house-style.md` の境界ケース規則による。
