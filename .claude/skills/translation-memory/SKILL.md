# translation-memory — 翻訳メモリ（TM・セグメント単位）

過去の「原文 → 訳文」の対を貯め、新規案件で似た箇所を再利用して一貫性と速度を上げる。
`glossary/`（語レベル）・`improvement/learnings.md`（ルールレベル）とは別の、**文・セグメントレベル**の記憶。

## 0. 最重要: 機密の扱い（learnings とは逆）

- TM は**生の対訳データ**（クライアントの原文・訳文そのもの）。マスキングすると再利用価値が消えるため、**マスキングしない**。
- そのかわり **完全にローカル限定**。実データは `tm/tm.local.jsonl` に保存し、配布・公開リポには**絶対に commit しない**（`.gitignore` 済み）。
- 共有が必要なら private な経路のみ（SPEC §7・§9-4）。
- 保存先がローカル限定であることを、追記時に利用者へ明示する。

## 1. 保存形式

`tm/tm.local.jsonl`（1 行 1 対訳・JSONL）:

```json
{"date":"2026-06-17","client":"実名でよい","domain":"分野","profile":"profile-*","source":"原文","target":"訳文","note":"補足"}
```

- `client` はローカル限定なので**実名で記録してよい**（git に乗らない）。クライアント別の一貫性確認に使う。
- ファイルが無ければ `tm/tm.example.jsonl` の形式に倣って新規作成する。

## 2. 翻訳前：参照（retrieve）

1. 今回の原稿から、固有名詞・専門用語・定型表現・繰り返し出る文を抜き出す。
2. `tm/tm.local.jsonl` を、今回の `client` やそれらの語句で検索する。
3. 過去訳が見つかれば**それに合わせて訳し、一貫性を保つ**。過去訳と今回の文脈が食い違う場合は、勝手に変えず利用者に確認する。

> 注: いまは語句一致・完全一致ベースの簡易検索。意味的なあいまい一致（fuzzy match）は将来拡張（embeddings か CAT ツール取り込み）。件数が増えたら検索範囲を `client` で絞る。

## 3. 翻訳後：追記（append）

1. 確定した「原文 → 訳文」の対を、**利用者の同意のうえ** `tm/tm.local.jsonl` に追記する。
2. 1 文ずつ、または意味のまとまりごとに 1 行で記録する。長文は段落単位でよい。
3. 機密案件でも TM はローカル限定なので保存してよい。ただし「これは手元の TM にだけ保存します」と明示する。

## 4. 他レイヤーとの分担

| レイヤー | 単位 | 置き場 | 機密 |
|---|---|---|---|
| 用語 | 語 | `glossary/` | 匿名化して共有可 |
| ルール・文体 | 規則 | `improvement/learnings.md` | 匿名化して共有可 |
| **対訳（本TM）** | **文・段落** | `tm/tm.local.jsonl` | **生データ・ローカル限定** |
